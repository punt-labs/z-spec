% ============================================================
\section{Introduction}
% ============================================================

This specification extends the base Claude Code state machine
(\texttt{claude-code.tex}) with vox text-to-speech notification state.
Vox provides spoken notifications, vibes (expressive mood tags), and
a decision-block pattern on the \texttt{Stop} hook that ensures Claude
speaks a summary before finishing each turn.

The key properties verified by this model:

\begin{enumerate}
  \item \textbf{No infinite Stop loop}: the decision-block pattern
    fires exactly once per turn. The \texttt{stopHookActive} flag
    prevents re-entry.
  \item \textbf{Speech requires notification}: \texttt{speakMode = ztrue}
    implies \texttt{notifyMode $\neq$ nmOff}. You cannot have voice
    without notifications.
  \item \textbf{Vibe signals accumulate monotonically}: signals from
    Bash tool output accumulate during a turn and are consumed at Stop.
  \item \textbf{Notification hooks are no-ops when disabled}: when
    \texttt{notifyMode = nmOff}, the Stop hook allows immediately
    (no block, no speech).
\end{enumerate}

\section{Vox Types}
% ============================================================

\begin{zed}
  NotifyMode ::= nmOff | nmOnDemand | nmContinuous
\end{zed}

Notification level. \texttt{nmOff} disables all notifications.
\texttt{nmOnDemand} fires on Stop and permission/idle prompts.
\texttt{nmContinuous} adds real-time signal announcements from a
background watcher.

\begin{zed}
  VibeMode ::= vmAuto | vmManual | vmOff
\end{zed}

How expressive tags are selected. \texttt{vmAuto} reads signals from
Bash output and resolves tags at Stop. \texttt{vmManual} uses explicit
user-provided tags. \texttt{vmOff} disables vibe entirely.

\begin{zed}
  [SIGNAL]
\end{zed}

A vibe signal token (e.g., \texttt{tests-pass}, \texttt{lint-fail},
\texttt{git-push-ok}). Accumulated during a turn from Bash tool output.

\section{State}
% ============================================================

\begin{schema}{State}
  %
  % --- Base Claude Code state ---
  %
  sessionId : \finset SESSIONID \\
  sessionPhase : SessionPhase \\
  permMode : PermissionMode \\
  toolPhase : ToolPhase \\
  currentTool : \finset TOOLNAME \\
  toolsThisTurn : \nat \\
  activeSubagents : \finset AGENTID \\
  context : \seq CONTEXTCHUNK \\
  pendingPrompt : \finset PROMPT \\
  pendingResponse : \finset RESPONSE \\
  %
  % --- Vox notification state ---
  %
  notifyMode : NotifyMode \\
  speakMode : ZBOOL \\
  vibeMode : VibeMode \\
  vibeSignals : \finset SIGNAL \\
  stopHookActive : ZBOOL \\
  spokeSummary : ZBOOL
  \where
  %
  % === Base invariants ===
  %
  \# sessionId \leq 1 \\
  sessionPhase = spInactive \implies sessionId = \emptyset \\
  sessionPhase \neq spInactive \implies \# sessionId = 1 \\
  sessionPhase \neq spProcessing \implies toolPhase = tpNone \\
  \# currentTool \leq 1 \\
  toolPhase = tpNone \implies currentTool = \emptyset \\
  toolPhase \neq tpNone \implies \# currentTool = 1 \\
  toolsThisTurn \leq maxTools \\
  sessionPhase = spInactive \implies activeSubagents = \emptyset \\
  sessionPhase = spEnded \implies activeSubagents = \emptyset \\
  \# activeSubagents \leq maxSubagents \\
  \# context \leq maxContext \\
  \# pendingPrompt \leq 1 \\
  sessionPhase = spIdle \implies pendingPrompt = \emptyset \\
  sessionPhase = spInactive \implies pendingPrompt = \emptyset \\
  sessionPhase = spEnded \implies pendingPrompt = \emptyset \\
  \# pendingResponse \leq 1 \\
  sessionPhase \neq spResponding \implies pendingResponse = \emptyset \\
  %
  % === Vox invariants ===
  %
  % Speech requires notification to be enabled
  %
  speakMode = ztrue \implies notifyMode \neq nmOff \\
  %
  % Vibe signals bounded for animation
  %
  \# vibeSignals \leq maxSignals \\
  %
  % Stop hook flags only during the respond-speak-respond cycle
  % (spProcessing with stopHookActive=ztrue is the speak phase)
  %
  sessionPhase = spIdle \implies stopHookActive = zfalse \\
  sessionPhase = spInactive \implies stopHookActive = zfalse \\
  sessionPhase = spEnded \implies stopHookActive = zfalse \\
  %
  sessionPhase = spIdle \implies spokeSummary = zfalse \\
  sessionPhase = spInactive \implies spokeSummary = zfalse \\
  sessionPhase = spEnded \implies spokeSummary = zfalse \\
  %
  % spokeSummary implies stopHookActive (spoke only after block)
  %
  spokeSummary = ztrue \implies stopHookActive = ztrue \\
  %
  % No vox state when inactive
  %
  sessionPhase = spInactive \implies vibeSignals = \emptyset \\
  sessionPhase = spInactive \implies notifyMode = nmOff
\end{schema}

\section{Response and Stop --- Decision-Block Pattern}
% ============================================================

This section models the vox Stop hook's re-entrant decision-block
pattern. The sequence is:

\begin{enumerate}
  \item Claude finishes reasoning $\to$ \texttt{BeginResponse}
  \item Claude streams response $\to$ attempts to stop
  \item \textbf{First Stop}: if notify enabled, hook blocks
    (\texttt{StopHookBlock}). Sets \texttt{stopHookActive = ztrue}.
    Claude re-enters processing to speak.
  \item Claude calls \texttt{unmute} to speak summary
    (\texttt{SpeakSummary}).
  \item Claude streams again $\to$ attempts to stop again
  \item \textbf{Second Stop}: hook sees \texttt{stopHookActive = ztrue},
    allows (\texttt{StopHookAllow}). Session returns to idle.
\end{enumerate}

\section{System Invariants}
% ============================================================

\begin{enumerate}
  \item \textbf{No infinite Stop loop}: \texttt{StopHookBlock} sets
    \texttt{stopHookActive = ztrue}. \texttt{StopHookAllow} requires
    \texttt{stopHookActive = ztrue}. There is exactly one block and
    one allow per turn. No path enables a second block.

  \item \textbf{Speech requires notification}: the state invariant
    \texttt{speakMode = ztrue $\implies$ notifyMode $\neq$ nmOff}
    holds across all reachable states. \texttt{DisableNotify} clears
    both. \texttt{EnableNotify} sets both. \texttt{Mute} clears speak
    but not notify.

  \item \textbf{Signal accumulation}: \texttt{AccumulateSignal} only
    fires when \texttt{vibeMode = vmAuto}. Signals are consumed by
    \texttt{SpeakSummary} (cleared to $\emptyset$).

  \item \textbf{stopHookActive only during responding}: the invariant
    \texttt{sessionPhase $\neq$ spResponding $\implies$
    stopHookActive = zfalse} ensures the flag does not leak across
    turns. \texttt{SubmitPrompt} resets it.

  \item \textbf{spokeSummary implies stopHookActive}: you can only
    have spoken a summary after the stop hook blocked. No path sets
    \texttt{spokeSummary = ztrue} without first setting
    \texttt{stopHookActive = ztrue}.

  \item \textbf{Notification disabled $\implies$ skip}: when
    \texttt{notifyMode = nmOff}, only \texttt{StopHookSkip} can fire
    (not \texttt{StopHookBlock}). The session transitions directly
    to idle with no speech.
\end{enumerate}
